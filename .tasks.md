# Development Tasks for AI Guitar Tutor

This document defines the development roadmap for the project.

The AI assistant should follow these tasks sequentially when generating code.

Each task should be completed and tested before moving to the next one.

---

# Phase 1 — Repository Setup

Tasks:

• Create project directory structure
• Add README.md
• Add .ai_rules.md
• Add .architecture.md
• Add .tasks.md

Verify repository structure:

frontend/
backend/
data/
infra/

---

# Phase 2 — Lesson Service

Objective:

Implement a service that loads chord lessons from disk.

Tasks:

• create Lesson dataclass
• implement LessonService
• load lesson metadata
• detect lesson directories automatically
• implement get_lesson(chord_name)

Testing:

Ensure lessons can be loaded from:

data/lessons/

---

# Phase 3 — Audio Evaluation Engine

Objective:

Create a chord evaluation system.

Tasks:

• implement ChordEvaluator class
• load audio file
• apply noise reduction
• compute FFT spectrum
• detect dominant frequencies
• map frequencies to musical notes
• compare notes with expected chord notes
• generate evaluation result

Testing:

Use recorded guitar chord samples.

Verify correct note detection.

---

# Phase 4 — Feedback Engine

Objective:

Convert evaluation results into tutor responses.

Tasks:

• create FeedbackGenerator class
• map issues to feedback templates
• support multiple error types

Examples:

muted_string
missing_note
wrong_chord

Testing:

Verify feedback generation from evaluation results.

---

# Phase 5 — Tutor Agent

Objective:

Implement session orchestration.

Tasks:

• create TutorAgent class
• implement session state machine
• integrate LessonService
• integrate ChordEvaluator
• integrate FeedbackGenerator

Testing:

Simulate learning session flow.

---

# Phase 6 — Backend API

Objective:

Expose backend services through REST API.

Tasks:

Create FastAPI server.

Endpoints:

POST /learn_chord
POST /submit_audio
GET /lesson/{chord}

Testing:

Use curl or Postman.

---

# Phase 7 — Frontend Interface

Objective:

Build user interface.

Tasks:

• create chord input field
• create lesson video player
• implement microphone recording
• send audio to backend
• display tutor feedback

Testing:

Verify full learning flow.

---

# Phase 8 — Local Deployment

Objective:

Run the system locally.

Tasks:

• create Dockerfiles
• create docker-compose configuration
• run frontend and backend together

---

# Phase 9 — Cloud Deployment

Objective:

Deploy system to Google Cloud.

Tasks:

• deploy backend to Cloud Run
• store lesson assets in Cloud Storage
• configure Firestore for user data
